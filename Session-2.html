Pourquoi faire des statistiques descriptives?

Les analyses descriptives constituent la premire tape de toute analyse quantitative. Elles permettent de dcrire les donnes, den reprer les caractristiques principales, didentifier dventuels problmes et dorienter les choix mthodologiques pour les analyses ultrieures.

Elles permettent de rpondre des questions simples mais essentielles telles que :

  • Quelles valeurs prennent les variables ?
  • Comment ces valeurs sont-elles rparties ?
  • Certaines valeurs sont-elles plus frquentes que dautres ?
  • Observe-t-on des valeurs manquantes ou inhabituelles ?

Avant de comparer des groupes ou destimer des modles statistiques, il est indispensable de bien comprendre les donnes avec lesquelles on travaille. Dans notre cas, il est utile de savoir quavant de chercher expliquer des phnomnes sociaux laide de donnes, il est indispensable de comprendre ce que sont ces donnes.

Trois raisons peuvent tre mentionnes pour justifier ces analyses:

1. Comprendre les donnes

Les statistiques descriptives permettent didentifier :

  • les valeurs typiques dune variable (par exemple la moyenne ou la mdiane),
  • le degr de dispersion des observations,
  • la forme gnrale de la distribution.

Par exemple, connatre lge moyen dun chantillon nest pas suffisant si lon ne sait pas si la majorit des individus ont un ge proche de cette valeur ou si les ges sont trs disperss.

2. Dtecter les erreurs et les problmes de donnes

Les analyses descriptives permettent souvent de reprer :

  • des erreurs de codage (et besoin dventuel recodage des donnes),
  • des valeurs impossibles ou incohrentes,
  • des valeurs extrmes,
  • des donnes manquantes.

Par exemple, une variable mesurant le nombre dannes dtudes ne devrait pas contenir la valeur 97. Ce type dinformation indique un problme corriger avant toute analyse plus approfondie.

3. Orienter les choix mthodologiques

Comprendre les donnes, leur nature et la distribution des variables permet de dterminer :

  • les statistiques pertinentes utiliser,
  • les graphiques appropris,
  • les mthodes statistiques mobilisables par la suite.

Les analyses descriptives ont des limites

Il est important de souligner que les analyses descriptives ne suffisent pas tout expliquer. Il faut diffrencier la description de donnes et lanalyse de donne. Les analyses descriptives ne permettent pas :

  • dexpliquer des relations entre variables,
  • dtablir des liens de causalit,
  • des tester des hypothses de recherche.

Elles servent prparer et clairer lanalyse, mais ne remplacent pas les mthodes dinfrence statistique.

Les variables catgorielles sont des variables qui indiquent des lments qui ne sont pas, ou difficilement quantifiables. Par exemple, le choix de vote dans la base de donnes BES ou le nom des pays dans la base de donnes country nest pas une variable qui a un ordre et peut tre mesure.

Recoder les variables dans une base de donnes.

Dans une base de donnes, il est frquent que certaines variables ne soient pas directement exploitables pour lanalyse. Elles peuvent tre codes sous une forme peu claire, trop dtaille, ou ne pas correspondre la question de recherche. Recoder une variable consiste transformer ses valeurs afin de les rendre plus adaptes lanalyse, tout en conservant linformation utile. En pratique, il est recommand de crer une nouvelle variable recode, plutt que de modifier la variable originale, afin de conserver les donnes de dpart.

Pourquoi recoder une variable ?

Le recodage permet notamment de :

  • Simplifier une variable trop dtaille
  • Regrouper des catgories similaires
  • Transformer une variable numrique en classes
  • Harmoniser des codages diffrents
  • Prparer les donnes pour un modle statistique

Pour comprendre et analyser ses donnes, savoir recoder une variable est un lment essentiel.

Comment recoder une variable?

En R, le recodage se fait en crant une nouvelle variable partir dune variable existante, laide de conditions.

La structure gnrale: - Si la valeur respecte une condition > alors on lui attribue une nouvelle valeur.

Concrtement, prenons lexemple de la variable age dans la base de donnes BES on peut crer des catgories pour diffrencier les personnes entre 18 et 40 ans, les personnes entre 41 et 64 ans et les personnes de 65 ans ou plus, on peut faire:

BES$age_classe <- ifelse(BES$age <= 40, "18-40", ### Si la variable age est infrieure ou gale  40, alors "18-40", sinon...
                          ifelse(BES$age>40 & BES$age<65, "41-64",  ### Si la variable age est entre 40 et 65 alors "41-64" sinon...
                                 "65+")) ### 65+ pour les cas restants

table(BES$age_class)
